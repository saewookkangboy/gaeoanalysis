## 개요

### Spec-Kit이란?

Spec-Kit은 AI 에이전트(Claude Code 등)와 함께 사용하여 **Spec-Driven Development**를 수행하는 도구입니다. 명세서를 먼저 작성하고, 이를 기반으로 체계적으로 개발을 진행합니다.

### 핵심 워크플로우

```
Constitution → Spec → Plan → Research → Validation → Tasks → Implementation
```

### 현재 프로젝트 상태

- ✅ 이미 잘 구조화된 프로젝트
- ✅ 명확한 기능 정의 (README, ARCHITECTURE 문서)
- ✅ 체계적인 문서화
- ⚠️ Spec-Driven Development 워크플로우 미적용
- ⚠️ 기능별 명세서 부재

## Phase 1: Constitution (헌법) 작성

### 목적
프로젝트의 기본 원칙, 코딩 스타일, 아키텍처 결정을 문서화합니다.

### 체크리스트

#### 1.1 프로젝트 원칙 정의
- [ ] **코드 품질 원칙**
  - [ ] TypeScript 엄격 모드 사용
  - [ ] ESLint 규칙 준수
  - [ ] 함수/컴포넌트 단일 책임 원칙
  - [ ] 에러 핸들링 필수
  - [ ] 타입 안정성 보장

- [ ] **아키텍처 원칙**
  - [ ] Next.js App Router 사용
  - [ ] Server Components 우선 사용
  - [ ] API Route는 Route Handler 패턴
  - [ ] 데이터베이스는 헬퍼 함수로 추상화
  - [ ] 비즈니스 로직은 `lib/` 디렉토리에 분리

- [ ] **보안 원칙**
  - [ ] 모든 사용자 입력 검증 (Zod)
  - [ ] 레이트 리미팅 적용
  - [ ] XSS 방지 (입력 sanitization)
  - [ ] 환경 변수로 민감 정보 관리
  - [ ] 보안 헤더 자동 설정

- [ ] **성능 원칙**
  - [ ] 메모리 캐싱 활용
  - [ ] 데이터베이스 쿼리 최적화 (인덱스)
  - [ ] 이미지/리소스 최적화
  - [ ] 레이지 로딩 적용

#### 1.2 코딩 컨벤션
- [ ] **네이밍 규칙**
  - [ ] 컴포넌트: PascalCase (예: `ScoreCard.tsx`)
  - [ ] 함수/변수: camelCase (예: `calculateScore`)
  - [ ] 상수: UPPER_SNAKE_CASE (예: `MAX_RETRY_COUNT`)
  - [ ] 타입/인터페이스: PascalCase (예: `AnalysisResult`)

- [ ] **파일 구조 규칙**
  - [ ] 한 파일에 하나의 주요 export
  - [ ] 관련 타입은 같은 파일 또는 `types/` 디렉토리
  - [ ] 유틸리티 함수는 `lib/` 디렉토리
  - [ ] 컴포넌트는 `components/` 디렉토리

- [ ] **커밋 메시지 규칙**
  - [ ] Conventional Commits 형식
  - [ ] `feat:`, `fix:`, `refactor:`, `docs:`, `style:`, `chore:` 사용
  - [ ] 자동 Git 푸시 (`.cursorrules`에 정의됨)

#### 1.3 문서화 원칙
- [ ] **코드 문서화**
  - [ ] 복잡한 함수는 JSDoc 주석
  - [ ] 타입 정의에 설명 추가
  - [ ] API 엔드포인트에 주석

- [ ] **프로젝트 문서화**
  - [ ] README.md 최신 상태 유지
  - [ ] ARCHITECTURE.md 아키텍처 문서
  - [ ] 기능별 상세 문서 (예: `ALGORITHM_LEARNING_SYSTEM.md`)

- 모든 입력 검증 (Zod)
- 레이트 리미팅 필수
- XSS 방지
