# 일반 사이트 강화 분석 기준 제안서

## 📋 개요

일반 사이트 도메인(블로그 제외)에 대해 AIO, GEO, AI SEO 측정을 강화하고, 콘텐츠 분석을 더욱 깊이 있게 수행하여 블로그 콘텐츠와 차별화된 분석 기준을 적용하는 방안을 제안합니다.

## 🎯 목표

1. **일반 사이트와 블로그의 명확한 차별화**
   - 블로그 플랫폼 자동 감지 및 분류
   - 일반 사이트 전용 강화 분석 모듈 구축

2. **강화된 분석 기준 적용**
   - AIO, GEO, AI SEO 점수 계산 기준 강화
   - 일반 사이트 특성에 맞는 가중치 및 보너스 시스템

3. **깊이 있는 콘텐츠 분석**
   - 일반 사이트의 구조적 특성 반영
   - 비즈니스/전문 사이트에 최적화된 분석 항목 추가

## 🔍 현재 상태 분석

### 현재 구조
- **블로그 감지**: 네이버 블로그만 별도 처리 (`isNaverBlog`)
- **일반 사이트 분석**: `calculateSEOScore()`, `calculateAEOScore()`, `calculateGEOScore()` 사용
- **블로그 분석**: `analyzeNaverBlogContent()` 전용 모듈 사용
- **AIO 점수**: 모든 사이트에 동일한 가중치 적용

### 문제점
1. 블로그 플랫폼 감지가 네이버 블로그에만 국한됨
2. 일반 사이트와 블로그의 분석 기준이 동일함
3. 일반 사이트의 구조적 특성(비즈니스, 전문성, 신뢰도)이 충분히 반영되지 않음
4. AIO 점수 계산 시 일반 사이트 특화 가중치가 없음

## 💡 제안 사항

### 1. 블로그 플랫폼 감지 시스템 확장

#### 1.1 블로그 플랫폼 감지 함수
```typescript
interface BlogPlatform {
  type: 'naver' | 'tistory' | 'brunch' | 'wordpress' | 'medium' | 'velog' | 'none';
  confidence: number; // 0-1
}

function detectBlogPlatform(url: string, html: string): BlogPlatform {
  // URL 패턴 기반 감지
  // HTML 메타데이터 기반 감지
  // 구조적 특징 기반 감지
}
```

#### 1.2 감지 대상 플랫폼
- **네이버 블로그**: `blog.naver.com`
- **티스토리**: `*.tistory.com`
- **브런치**: `brunch.co.kr`
- **워드프레스**: `*.wordpress.com`, `*.wp.com`
- **Medium**: `medium.com`
- **Velog**: `velog.io`
- **기타**: 메타데이터 기반 감지 (generator 태그 등)

### 2. 일반 사이트 전용 강화 분석 모듈

#### 2.1 모듈 구조
```
lib/
  ├── website-analyzer.ts          # 일반 사이트 전용 분석 모듈
  ├── blog-detector.ts             # 블로그 플랫폼 감지
  ├── enhanced-scoring.ts         # 강화된 점수 계산
  └── content-depth-analyzer.ts    # 깊이 있는 콘텐츠 분석
```

#### 2.2 일반 사이트 특화 분석 항목

**A. 비즈니스 신뢰도 지표**
- 회사 정보 완성도 (회사 소개, 연락처, 주소)
- 법적 정보 (이용약관, 개인정보처리방침)
- 인증/수상 내역
- 고객 후기/리뷰 시스템
- 보안 인증 (SSL, 보안 뱃지)

**B. 전문성 지표**
- 작성자/팀 정보 및 자격증명
- 전문 용어 사용 및 정의
- 연구/데이터 기반 콘텐츠
- 출처 명시 및 인용 품질
- 업데이트 주기 및 최신성

**C. 구조적 완성도**
- 네비게이션 구조
- 사이트맵 존재 여부
- 검색 기능
- 다국어 지원
- 접근성 (WCAG 준수)

**D. 콘텐츠 품질**
- 콘텐츠 길이 및 깊이 (일반 사이트는 더 긴 콘텐츠 기대)
- 섹션 구조 및 계층
- 미디어 활용 (이미지, 비디오, 인포그래픽)
- 상호작용 요소 (폼, 계산기, 도구)

### 3. 강화된 점수 계산 기준

#### 3.1 SEO 점수 강화 (일반 사이트)

**기존 기준 (100점)**
- H1 태그: 20점
- Title 태그: 15점
- Meta description: 15점
- Alt 텍스트: 10점
- 구조화된 데이터: 10점
- 기타: 30점

**강화 기준 (120점 만점)**
- 기존 항목 유지
- **추가 항목 (+20점)**:
  - 사이트맵 존재: 5점
  - robots.txt 존재: 3점
  - Breadcrumb 구조: 4점
  - 다국어 메타데이터: 3점
  - Open Graph 완성도: 5점

#### 3.2 AEO 점수 강화 (일반 사이트)

**기존 기준 (100점)**
- 질문 형식: 20점
- FAQ 섹션: 15점
- 답변 구조: 20점
- 키워드 밀도: 10점
- 기타: 35점

**강화 기준 (130점 만점)**
- 기존 항목 유지
- **추가 항목 (+30점)**:
  - 전문가 Q&A 섹션: 10점
  - 단계별 가이드 완성도: 8점
  - 비교표/대안 제시: 7점
  - 사례 연구(Case Study): 5점

#### 3.3 GEO 점수 강화 (일반 사이트)

**기존 기준 (100점)**
- 콘텐츠 길이: 20점
- 다중 미디어: 15점
- 섹션 구조: 15점
- 키워드 다양성: 15점
- 기타: 35점

**강화 기준 (140점 만점)**
- 기존 항목 유지
- **추가 항목 (+40점)**:
  - 포괄적 콘텐츠 깊이 (2000+ 단어): 10점
  - 전문 데이터/통계 포함: 8점
  - 인포그래픽/차트: 7점
  - 비디오 콘텐츠: 8점
  - 다국어 콘텐츠: 4점
  - 업데이트 주기 명시: 3점

### 4. AIO 점수 강화 (일반 사이트 특화 가중치)

#### 4.1 일반 사이트 특화 AIO 가중치

**기존**: 모든 사이트에 동일한 가중치
**강화**: 일반 사이트에 더 높은 기준 적용

```typescript
interface EnhancedAIOWeights {
  // ChatGPT 가중치 강화
  chatgpt_authority_weight: 0.15;  // 기존 0.10 → 0.15
  chatgpt_structure_weight: 0.20;   // 기존 0.15 → 0.20
  
  // Perplexity 가중치 강화
  perplexity_freshness_weight: 0.18; // 기존 0.12 → 0.18
  perplexity_citation_weight: 0.15;  // 기존 0.10 → 0.15
  
  // Claude 가중치 강화
  claude_depth_weight: 0.25;         // 기존 0.20 → 0.25
  claude_source_weight: 0.20;         // 기존 0.15 → 0.20
}
```

#### 4.2 일반 사이트 특화 보너스

**ChatGPT 보너스 강화**
- 전문가 자격증명: +8점 (기존 +6점)
- 연구 기반 콘텐츠: +7점 (신규)
- 비즈니스 인증: +5점 (신규)

**Perplexity 보너스 강화**
- 최신 업데이트 (30일 이내): +18점 (기존 +15점)
- 출처 링크 10개 이상: +10점 (기존 +7점)
- 데이터/통계 포함: +8점 (신규)

**Claude 보너스 강화**
- 주요 출처 5개 이상: +15점 (기존 +12점)
- 3000+ 단어 포괄적 콘텐츠: +12점 (기존 +10점)
- 방법론 명시: +10점 (기존 +8점)

### 5. 깊이 있는 콘텐츠 분석

#### 5.1 콘텐츠 구조 분석

**섹션 계층 분석**
- H1 → H2 → H3 → H4 계층 구조 완성도
- 섹션별 콘텐츠 밀도
- 섹션 간 연결성 (내부 링크)

**콘텐츠 타입 분석**
- 정보 제공형 콘텐츠
- 가이드/튜토리얼
- 비교/리뷰
- 뉴스/업데이트
- FAQ/지원

#### 5.2 신뢰도 신호 분석

**E-E-A-T 강화 분석**
- **Experience**: 실제 경험 기반 콘텐츠
- **Expertise**: 전문성 증명 (자격, 경력)
- **Authoritativeness**: 권위성 (인용, 수상)
- **Trustworthiness**: 신뢰성 (보안, 투명성)

**분석 항목**
- 작성자 프로필 완성도
- 회사/조직 정보
- 인증/수상 내역
- 고객 후기/리뷰
- 보안 인증

#### 5.3 상호작용 요소 분석

- 폼/문의 양식
- 계산기/도구
- 댓글 시스템
- 소셜 공유 기능
- 구독/알림 기능
