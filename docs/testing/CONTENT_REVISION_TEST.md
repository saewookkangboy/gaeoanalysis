# 콘텐츠 수정안 미리보기 테스트 가이드

## 테스트 URL
- **네이버 블로그**: https://blog.naver.com/mercyblu/224144269771

## 테스트 절차

### 1. 환경 준비
```bash
# 개발 서버 실행
npm run dev
```

### 2. 브라우저에서 테스트

1. **로그인**
   - http://localhost:3000 접속
   - "로그인" 버튼 클릭
   - Google 또는 GitHub로 로그인

2. **분석 진행**
   - URL 입력 필드에 `https://blog.naver.com/mercyblu/224144269771` 입력
   - "분석 시작" 버튼 클릭
   - 분석 완료 대기 (약 30초)

3. **콘텐츠 수정안 미리보기**
   - 분석 결과 페이지에서 "콘텐츠 수정안 미리 보기 (개발 모드)" 버튼 확인
   - 버튼 클릭
   - "미리보기 생성" 버튼 클릭

4. **결과 확인**
   - ✅ 텍스트 중심으로 표시되는지 확인
   - ✅ HTML/마크다운 문법이 제거되었는지 확인
   - ✅ 원문 구조가 유지되는지 확인
   - ✅ 복사 기능 작동 확인
   - ✅ 예상 점수 변화 확인

## 예상 결과

### 텍스트 중심 출력
- HTML 태그 없음
- 마크다운 문법 없음
- 순수 텍스트만 표시
- 블로그 플랫폼에 바로 붙여넣기 가능

### 원문 구조 유지
- 제목 구조 유지
- 문단 순서 유지
- 리스트 구조 유지

### 개선 포인트
- SEO 개선 사항 반영
- AEO 개선 사항 반영
- GEO 개선 사항 반영

## 문제 해결

### 버튼이 보이지 않는 경우
- 개발 모드(`NODE_ENV=development`)에서만 표시됨
- 프로덕션에서는 숨김 처리됨

### 로그인 오류
- Firebase 설정 확인
- OAuth 설정 확인

### 미리보기 생성 실패
- Gemini API 키 확인
- 네트워크 연결 확인
- 콘솔 로그 확인
